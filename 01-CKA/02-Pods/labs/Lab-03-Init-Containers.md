# What is an Init Container?

An Init Container runs **before** the application containers.

It must complete successfully before Kubernetes starts the main application.

Characteristics:

* Runs once
* Runs sequentially
* Must exit successfully
* Cannot run alongside application containers

---

# Production Use Cases

Init Containers are commonly used for:

* Waiting for a database
* Downloading configuration
* Running database migrations
* Preparing application files
* Generating certificates
* Verifying dependencies

# Step 1 - Create the YAML

Create:

```text
init-container.yaml
```

Paste:

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: init-container-demo

spec:

  volumes:
    - name: shared-data
      emptyDir: {}

  initContainers:

    - name: init-container

      image: busybox

      command:
        - sh
        - -c
        - |
          echo "Welcome to Kubernetes!" > /data/index.html
          echo "Generated by Init Container" >> /data/index.html

      volumeMounts:
        - name: shared-data
          mountPath: /data

  containers:

    - name: nginx

      image: nginx

      volumeMounts:
        - name: shared-data
          mountPath: /usr/share/nginx/html
```

# Step 2 - Create the Pod

```bash
kubectl apply -f init-container.yaml
```

Verify:


# Step 3 - Observe Init Container

Describe the Pod:

```bash

```

Observe:

* Init Containers
* Main Containers
* Events

Notice that the Init Container completed before NGINX started.

# Step 4 - Check Init Container Logs


# Step 5 - Verify Generated File

Open a shell:

```bash
kubectl exec -it init-container-demo -- sh
```

Verify:

```bash

```

Expected output:

```text
Welcome to Kubernetes!
Generated by Init Container
```

Exit:

```bash

```

# Common Errors

## Init Container Never Completes

Check:

```bash
kubectl logs init-container-demo -c init-container

kubectl describe pod init-container-demo
```

---

## Main Container Never Starts

Remember:

Application containers wait until every Init Container completes successfully.

---

## Shared Volume Missing

Check:

```bash
kubectl describe pod init-container-demo
```

Verify:

* Volumes
* Volume Mounts

# Production Discussion

Good use cases:

* Database migration
* Downloading configuration
* Waiting for dependent services
* Certificate generation
* Application initialization

Poor use cases:

* Long-running monitoring
* Log collection
* Service mesh proxies

Those belong in Sidecar Containers.
